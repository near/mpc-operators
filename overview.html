NEAR MPC — Network & Security Overview

How the NEAR Multi-Party Computation (MPC) signing network is structured, secured, and independently verifiable.

→ View the live operator set

NEAR's MPC network (Chain Signatures) lets a NEAR account sign transactions on other chains with no single party ever holding the private key. The key is split across independent operators, and a signature is valid only when a threshold of them cooperate — so there is no single key and no single operator that can sign alone, and the full key never exists in one place.

Who operates the nodes?

Independent professional infrastructure providers — not the NEAR Foundation or NEAR Inc. The set currently includes entities such as Aurora, Everstake, Luganodes, Stakin, Black Sand, Node Monster, Staking4All, MPCH, Sovereign and NEAR One. The live operator page lists the current members with their NEAR account IDs and node endpoints.

How are the key shares generated?

Through a fully distributed key generation (DKG) protocol (a PedPop variant) — no trusted dealer. Each operator generates its own share and the full private key is never assembled on any machine, at any point. When operators join or leave, shares are reshared to the new set (the threshold can change) without changing the public key or exposing the key. DKG documentation →

What protects a node while it runs?

Nodes run inside Intel TDX confidential VMs (TEEs), isolating each key share from the host OS and the operator itself. Every node publishes an on-chain-verifiable remote attestation and runs only reproducible, on-chain-approved software, so anyone can confirm a node runs the expected, unmodified code. This also enforces backward secrecy: a departing operator cannot retain usable key material. TEE enforcement is being rolled out across the network, with attestation status tracked on-chain.

How does the network change (governance)?

Adding or removing an operator, or changing the threshold, is an on-chain vote by the existing operators. Only after enough current operators approve does the network reshare to the new set. No single party can alter the network unilaterally.

What if operators go offline or collude?

Signing continues as long as at least the threshold number of operators are available; below that, the worst case is a failed signature — not a key compromise. Forging a signature or recovering the key would require colluding across at least a threshold of independent operators and defeating the TEE on each — the core security assumption of the system.

Signing schemes

  • ECDSA / secp256k1 — Bitcoin, Ethereum & EVM chains.
  • EdDSA / Ed25519 — Solana and other Ed25519 chains.
  • Confidential key derivation / BLS12-381.

Verify it yourself

The participant set and threshold are public on-chain. Query the state method of v1.signer:

near contract call-function as-read-only v1.signer \
  state json-args {} network-config mainnet now

Or in the explorer: nearblocks.io/address/v1.signer (call state under Contract). The live operator page renders the same data as a table.
